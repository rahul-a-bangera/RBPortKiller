- **Development Guide**: Misc/Docs/DEVELOPMENT.md`n- **Architecture**: Misc/Docs/ARCHITECTURE.md`n- **Usage**: Misc/Docs/USAGE.md`n
## Quick Start

1. Fork and clone the repository
2. Create a feature branch
3. Make changes following SOLID principles
4. Test thoroughly
5. Submit a pull request

## Code Standards

- Follow SOLID principles
- Use proper namespaces and file organization
- Add XML documentation for public APIs
- One class per file
- Constructor injection for dependencies

## Documentation

All documentation files must be placed in Misc/Docs/ directory.

## Testing

Use TestPortOpener utility for manual testing. See TestPortOpener/README.md.
